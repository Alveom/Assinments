Frequently Asked Questions

1. The difference between var, let, and const?
var, let, and const are used to declare variables in JavaScript. var is function-scoped and can be redeclared, let is block-scoped and can't be redeclared in the same scope, and const is also block-scoped but can't be reassigned after initialization
2. The difference between map(), forEach(), and filter()?
map(), forEach(), and filter() are array methods. map() creates a new array with transformed elements, forEach() loops through an array but doesn't return anything, and filter() creates a new array with elements that meet a condition.
3. Explain arrow functions and how they are different from regular functions
Arrow functions are a shorter way to write functions using =>. They don't have their own this value, meaning they inherit this from their surrounding scope, unlike regular functions which have their own this.
4. How JavaScript Promises work
JavaScript Promises handle asynchronous operations. A Promise has three states: pending, fulfilled, or rejected. It helps avoid callback hell by allowing .then() for success and .catch() for errors.
5. how closures work in JavaScript
Closures happen when a function remembers variables from its parent scope even after the parent function has finished running. This allows functions to keep private variables and create encapsulated logic.
